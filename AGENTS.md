# OpenCode Instructions - Publication System Frontend

## Toolchain & Commands
- **Package Manager**: Exclusively use `bun` (e.g., `bun install`, `bun run dev`, `bun run test`). Do not use `npm`, `yarn`, or `pnpm`.
- **Linting & Formatting**: This project uses **Biome**, not ESLint/Prettier. 
  - Run `bun run check` to verify linting and formatting.
  - Run `bun run format` to autofix formatting.
- **Testing**: Vitest with React Testing Library. Execute tests via `bun run test`.
- **Pre-commit Hooks**: Husky runs `bunx lint-staged` -> `bun check --fix`. Tests and linting must pass before committing.

## Architecture & Framework Quirks
- **Routing (`@tanstack/react-router`)**: File-based routing rooted at `src/app/routes/`.
  - **CRITICAL**: `src/routeTree.gen.ts` is auto-generated by the Vite plugin. **Never edit it manually.**
- **Project Structure (Bulletproof React Style)**:
  - `src/features/`: Domain-specific components, hooks, and logic (e.g., `auth`, `articles`, `accounts`).
  - `src/components/ui/`: Reusable, generic UI components (Shadcn/UI).
- **Styling (Tailwind CSS v4)**: 
  - There is **no** `tailwind.config.js` or `tailwind.config.ts`.
  - Configuration and variables are managed via the `@tailwindcss/vite` plugin and `src/styles.css`.
- **API Fetching**: 
  - Uses TanStack Query combined with an `ofetch` wrapper located at `src/lib/api-client.ts` (`export const api = ...`).
  - **Note**: The global `api` client already intercepts errors and triggers `sonner` toasts for standard HTTP codes (401, 403, 404, etc.). Avoid writing redundant `try/catch` toast logic in components unless custom error handling is required.
- **Forms**:
  - Uses `@tanstack/react-form` with Zod schemas colocated with mutations (e.g. `create-article.ts`, `update-account.ts`, `src/lib/auth.tsx` for login/register). Validators use Zod’s Standard Schema support.
- **Access Control (RBAC/ABAC)**:
  - This frontend strongly features Role-Based and Attribute-Based Access Control.
  - Core logic and hooks for permissions reside in `src/lib/authorization.tsx` and `src/utils/permissions.ts`. Refer to these files when modifying user access flows.

## Imports & Path Aliases
- `@/*` maps to `./src/*`.
- When modifying or using Shadcn components, they are referenced via `@/components/ui/*`.
- Utility functions like Tailwind merge (`cn`) are located at `src/utils/cn.ts` (aliased as `@/utils/cn` via `components.json`).
